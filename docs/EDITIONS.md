# ONE CMS — 無料 / Pro 機能線引き

> 社内向けプロダクト戦略メモ。エディション構成・機能配分・ロードマップの基準。

## 基本方針（線引きの原則）

- **無料 = それ単体で公開サイトを運用できる完全な CMS**（オフライン・単一サイト・記事数の人為制限なし）。OSS（MIT）なので魅力を保つ。
- **Pro = 大規模運用・行政品質要件・運用効率・ブランディング**に効く追加価値。コピーされても陳腐化する **継続更新 + サポート** を軸に守る。
- 「無料の便利機能を Pro 化」はしない（公開済み MIT のため反感を買い、かつ無意味）。Pro は **新規・上積み** で差別化する。
- **両エディションともオフライン厳守**（外部通信が必要な機能は入れない）。
- Pro は **別ビルド（`VITE_EDITION=pro`）** のプロプライエタリ。同一バイナリ内のフラグ制御に依存しない。

| 予約公開（未来日で自動非公開 → 当日公開）                 | 広報の定時公開運用                                 | ◯（書き出し時の現在日で判定） |

| 一括操作（一括ステータス変更・一括削除）                  | 運用効率                                           | ◯                             |
| リンク切れ・未翻訳チェック                                | 品質担保                                           | ◯（生成物解析）               |
| 操作履歴（監査ログ）／自動バックアップ世代管理            | 行政の説明責任                                     | ◯（`.cms` に記録）            |

## 商用・サポート層（コピー不能 = 最強の防御）

| 項目       | 無料                       | Pro / 有料                                  |
| ---------- | -------------------------- | ------------------------------------------- |
| ライセンス | MIT                        | 商用 EULA + ライセンス ID 透かし（実装済）  |
| 更新提供   | 自己責任（公開リポジトリ） | Pro 機能・テンプレの継続更新を購入者へ配布  |
| サポート   | コミュニティ／なし         | 導入支援・研修・保守契約・優先サポート・SLA |
| 配布       | 公開                       | 購入者限定の署名付き配布                    |

## 推奨ロードマップ（価値 × 工数）

- **Phase 1（即効・行政に刺さる／工数小〜中）**：アクセシビリティ点検、CSV インポート/エクスポート、予約公開
- **Phase 2（運用効率／中）**：複数サイト管理、一括操作、リダイレクト・高度 SEO、無制限リビジョン + 監査ログ
- **Phase 3（ブランディング・収益拡張）**：ホワイトラベル、プレミアムテンプレ、ネスト/フレキシブルブロック

## 注意点（線引きの落とし穴）

- **無料を「使えない版」にしない**：記事数・サイト数の人為制限や広告挿入は、OSS では即フォークされる。制限ではなく **上積み** で差別化。
- **Pro もオフライン厳守**：承認フローや予約公開も「外部サーバー前提」にしない（書き出し時判定・ファイルベースで実装）。
- **同一バイナリ内ゲートに頼らない**：Pro 機能は Pro ビルド（別配布）に含める（`VITE_EDITION=pro` で分岐）。
- **MIT 巻き戻し不可**：現行公開分は無料のまま。Pro は新規コードをプロプライエタリで。

## 関連する実装状況（2026-06 時点）

- エディション切替：`VITE_EDITION=pro`（`src/constants.ts` の `EDITION`）
- ライセンス透かし：`VITE_LICENSE_ID=<顧客ID>` で書き出し HTML に generator メタ + ID を注入（`src/export.ts`）
- 継続更新の基盤：データ移行（`src/migrations.ts`）・既定テンプレートの差分提案アップデート（`src/cms/output.ts`）
- 大規模対応：一覧の軽量インデックス + ページング、書き出しの並列化・増分化、検索インデックスの言語別分割
- 拡張ポイント：`src/cms/extensions.ts`（`extensionMixin` / `extensionNavItems` / `runExtensionInit`）。Pro を**フォークせず上積み**するための差し込み口。詳細は `docs/PRO-OVERLAY.md`

## Pro 開発のリポジトリ構成

- 無料コア = この公開リポ（MIT）。**Pro ソースは置かない**。
- Pro = **非公開リポ**で開発。無料コアを git submodule で取り込み、`src/cms/extensions.ts` と `index.html` を上書きして `VITE_EDITION=pro` でビルド（手順は `docs/PRO-OVERLAY.md`）。
